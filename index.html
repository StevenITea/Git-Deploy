📘 Tutoriel : créer et déployer un site simple sur GitHub Pages

Ce tutoriel a été réalisé par StevenITea.
Pour toute question ou demande d’aide : Discord : stevenltea

Objectif du tutoriel : apprendre à publier un site web simple à partir d’un fichier index.html et d’un fichier style.css, sans entrer dans des notions avancées de développement. L’idée est de se concentrer sur les bases : créer le projet, le lier à GitHub, envoyer les fichiers, puis activer GitHub Pages.

Selon le type de projet, il existe d’autres solutions comme MkDocs pour la documentation technique, Jekyll pour les sites statiques compatibles GitHub Pages, Hugo pour les sites rapides, ou encore WordPress pour les sites dynamiques. Ici, nous choisissons volontairement une approche simple en HTML/CSS pour bien comprendre la logique du déploiement.

🧩 Étape 1 : préparer les outils nécessaires

Avant de commencer, il faut disposer de deux éléments :

  • un compte GitHub ;
  • Visual Studio Code installé sur votre ordinateur.

Sur Windows, il est conseillé de lancer Visual Studio Code en administrateur lors de la première configuration, afin d’éviter certains problèmes liés aux droits d’accès.

📁 Étape 2 : créer le repository GitHub

Connectez-vous à GitHub puis créez un nouveau repository. Il est recommandé de choisir un nom court, de préférence en anglais, ainsi qu’une description claire. Vous pouvez également cocher l’option permettant d’ajouter un README, mais ce n’est pas obligatoire pour ce tutoriel.

🗂️ Étape 3 : créer le dossier du projet sur votre ordinateur

Créez un dossier à l’endroit de votre choix, par exemple ProjetGit. Ensuite, faites un clic droit sur ce dossier puis choisissez Ouvrir avec Code pour l’ouvrir directement dans Visual Studio Code.

Dans ce projet, la structure de base ressemblera à ceci : 

ProjetGit/
├── index.html
├── index-en.html
├── style.css
└── images/

Le dossier images contiendra les captures ou illustrations utilisées dans le site. GitHub Pages a besoin d’un fichier index.html pour afficher correctement la page web.

⚙️ Étape 4 : initialiser Git et configurer votre identité

Une fois le dossier ouvert dans Visual Studio Code, ouvrez un terminal via Terminal → Nouveau Terminal. Vous allez ensuite initialiser Git dans le dossier du projet et configurer votre identité GitHub. 

git init
git config user.name "votre pseudo github"
git config user.email "votre adresse mail github"

Si vous souhaitez appliquer cette configuration à tous vos futurs projets, vous pouvez utiliser l’option --global.

🔗 Étape 5 : récupérer le lien du repository GitHub

Retournez sur GitHub, ouvrez votre repository puis cliquez sur le bouton Code afin de récupérer l’URL du dépôt. Si le repository est vide, ce lien apparaît directement sur la page.

Cette URL permettra de relier votre dossier local à votre dépôt distant sur GitHub. 

git remote add origin https://github.com/USER/NOM-DU-REPO.git

Cette commande lie le projet local au dépôt GitHub distant. Si un ancien dépôt est déjà lié, vous pouvez le supprimer avec : 

git remote remove origin

Puis refaire la commande précédente avec la bonne URL. Pour vérifier le dépôt actuellement lié, utilisez : 

git remote -v

🖋️ Étape 6 : créer les fichiers du site

La prochaine étape consiste à créer le contenu du site. Pour cet exemple, nous utiliserons uniquement :

  • index.html pour la structure de la page ;
  • index-en.html pour la version anglaise ;
  • style.css pour la mise en forme ;
  • un dossier images pour les illustrations.

Pour cet exemple, le contenu du site a été préparé à partir d’un tutoriel existant, puis retravaillé avec l’aide d’une IA, afin d’obtenir une structure simple, claire et facile à comprendre pour un premier déploiement.

📤 Étape 7 : ajouter les fichiers et faire le premier commit

Une fois vos fichiers créés, vous devez les ajouter à Git pour préparer l’envoi vers GitHub. 

git add .

Cette commande ajoute tous les fichiers du projet, par exemple :

  • index.html
  • index-en.html
  • style.css
  • les images
  • et les autres fichiers éventuels

Ensuite, on crée un premier commit pour enregistrer cet état du projet : 

git commit -m "Ajout du site"

Le message du commit sert simplement à décrire les modifications effectuées. Vous pouvez choisir un autre message si besoin.

🌿 Étape 8 : envoyer le projet sur la branche principale

GitHub utilise généralement la branche main comme branche principale. On s’assure donc que le projet est bien placé dessus, puis on envoie le contenu sur GitHub. 

git branch -M main
git push -u origin main

À ce stade, votre code est maintenant présent sur GitHub.

🌍 Étape 9 : activer GitHub Pages

Une fois les fichiers envoyés, il ne reste plus qu’à publier le site. Pour cela, sur GitHub, allez dans :

  • Settings
  • Pages
  • Source → Deploy from a branch
  • Branch → main
  • Folder → / (root)
  • puis cliquez sur Save

GitHub générera alors une URL de publication sous la forme : 

https://tonpseudo.github.io/nom-du-repo/

C’est cette adresse que vous pourrez partager pour accéder à votre site en ligne.

🧪 Captures complémentaires du tutoriel

Voici la suite logique du tutoriel avec les nouvelles captures ajoutées à la fin du document, dans l’ordre d’utilisation. Elles montrent la création des fichiers, l’envoi vers GitHub, le cas du README déjà présent en ligne, puis l’activation de GitHub Pages.

1. Vérifier la création des fichiers du projet

Une fois vos fichiers créés, vous devez voir dans Visual Studio Code votre dossier ProjetGit avec index.html, style.css et le dossier images.

2. Ajouter les fichiers et faire le premier commit

Dans le terminal, vous pouvez ensuite lancer les commandes suivantes pour initialiser Git, ajouter les fichiers, créer le commit, puis renommer la branche en main. 

git init
git add .
git commit -m "Ajout du site"
git branch -M main

3. Cas particulier : un README existe déjà sur GitHub

Si vous avez créé un README directement sur GitHub, il se peut que votre dépôt distant contienne déjà un fichier que vous n’avez pas encore en local. Dans ce cas, un git pull peut ouvrir un message de fusion.

Il faudra alors valider le message de merge pour terminer l’opération, puis continuer normalement.

4. Envoyer le projet sur GitHub

Une fois le dépôt correctement relié, vous pouvez envoyer le projet avec : 

git push -u origin main

5. Vérifier que les fichiers sont bien présents sur le dépôt

Après l’envoi, vous devez retrouver vos fichiers sur GitHub : le dossier images, index.html, style.css et éventuellement le README.

6. Ouvrir les paramètres du repository

Pour publier le site, rendez-vous ensuite dans l’onglet Settings de votre repository.

7. Activer GitHub Pages

Dans la section Pages, choisissez Deploy from a branch. Si GitHub Pages est encore désactivé, vous verrez d’abord que la branche est définie sur None.

Sélectionnez ensuite la branche main et le dossier / (root), puis cliquez sur Save.

8. Vérifier le déploiement

Une fois GitHub Pages activé, GitHub affiche que le site est en cours de génération ou déjà déployé. Vous pouvez aussi voir le déploiement dans la page principale du repository.

✅ Résultat

Vous avez maintenant un site simple hébergé sur GitHub Pages à partir d’un projet HTML/CSS. Cette méthode est idéale pour publier un petit site vitrine, un tutoriel, une documentation simple ou un projet d’apprentissage.
